# 星汉灿烂 1v1 模式拓展包

`xinghan_canlan` 是基于新月杀框架开发的 1v1 竞技模式拓展，包含星汉灿烂专用游戏模式、规则技能、选将/上阵 QML 界面，以及一套 108 张专用牌堆。

## 功能概览

- 2 人专用 1v1 模式，固定一号位为主公（先手）、二号位为内奸（后手）。
- 开局展示 18 名武将，按“后 1、先 2、后 1”禁将。
- 禁将后按“后 1、先 2、后 2、先 2、后 2、先 2、后 2、先得剩余 1”的顺序构建双方武将池。
- 每个小局可派出 1 至 2 名武将；胜方武将锁定，败方武将回到武将池。
- 小局结束后进入重整流程，双方弃置区域内牌，重新派将，轮流摸起始手牌，并交换座位以交替行动顺序。
- 先赢得 3 个小局的一方获得最终胜利。
- 牌堆第 2 次洗牌后进入鏖战，【桃】可按【酒】响应；第 3 次洗牌后，当前回合角色在回合结束失去 1 点体力。
- QML 选将和上阵窗口支持更宽范围的房间尺寸，卡牌网格会按可用宽度自适应列数。

## 安装

1. 将整个 `xinghan_canlan` 文件夹复制到新月杀的 `packages` 目录。
2. 重启新月杀客户端。
3. 在游戏模式中选择“星汉灿烂”。

目录应类似：

```text
FreeKill/
└── packages/
    └── xinghan_canlan/
        ├── init.lua
        ├── pkg/
        ├── qml/
        └── i18n/
```

## 规则流程

### 1. 身份与先后手

模式固定 2 人开局。一号位为主公（先手），二号位为内奸（后手），身份开局即明置。每个小局结算后，系统会交换双方座位，让行动顺序在小局间交替。

### 2. 禁将

系统从公共武将池中展示 18 名武将：

1. 后手方禁用 1 名。
2. 先手方禁用 2 名。
3. 后手方禁用 1 名。

如果对话超时或返回异常，系统会按当前展示顺序自动补足应选数量。

### 3. 选将

禁将完成后，双方按以下顺序选择武将：

1. 后手方选择 1 名。
2. 先手方选择 2 名。
3. 后手方选择 2 名。
4. 先手方选择 2 名。
5. 后手方选择 2 名。
6. 先手方选择 2 名。
7. 后手方选择 2 名。
8. 先手方获得最后剩余的 1 名。

最终双方各获得 7 名武将。

### 4. 上阵与重整

每个小局开始时，双方从自己的武将池中选择 1 至 2 名武将上场。双将体力为两名武将体力之和的一半，向下取整。

小局结束后：

- 胜方本小局上场武将进入锁定区，后续不可再次上阵。
- 败方本小局上场武将回到自己的武将池。
- 双方弃置手牌、装备区与判定区的牌。
- 双方重新选择可用武将上阵。
- 双方按座位顺序轮流各摸 1 张牌，直到达到起始手牌数。
- 系统交换座位并继续下一小局。

## 牌堆

模式会使用 `pkg/xinghan_cards/init.lua` 中定义的专用 108 张牌堆。该牌堆只声明卡牌规格，卡牌类型来自标准牌堆和军争包。

牌堆包括：

- 基本牌：杀、火杀、雷杀、闪、桃、酒。
- 普通锦囊：决斗、过河拆桥、顺手牵羊、无中生有、借刀杀人、火攻、铁索连环、无懈可击等。
- 延时锦囊：闪电、乐不思蜀、兵粮寸断。
- 装备：常见武器、防具与坐骑。

## 本次优化

- 修复规则状态使用模块级全局表导致的潜在跨房间/连续开局串场问题。
- 修复禁将、选将和上阵对话超时后重复选择同一武将的问题。
- 修复武将池和锁定区在同名武将场景下可能重复或误删的问题。
- 修复先手摸牌惩罚可能把摸牌数减成负数的问题。
- 修复鏖战【桃】当【酒】未识别框架内部 `analeptic` 名称的问题。
- 修复第 3 次洗牌后回合结束扣体力可能作用到非当前回合角色的问题。
- 优化重整后起始手牌发牌顺序，使双方按座位轮流摸牌。
- 增加副将登场事件触发，避免双将场景只触发主将。
- 优化 QML 窗口尺寸、滚动条方向和卡牌网格列数，提升窄窗口适配。
- 修正牌堆注释与 README 中的数量和规则描述。

## 开发说明

- 游戏模式入口：`pkg/xinghan_mode/xinghan_1v1.lua`
- 规则技能：`pkg/xinghan_mode/rule_skills/xinghan_1v1.lua`
- 专用牌堆：`pkg/xinghan_cards/init.lua`
- 禁将/选将界面：`qml/XinghanSelect.qml`
- 上阵界面：`qml/XinghanDeploy.qml`

本仓库没有独立运行的新月杀框架依赖，因此本地只能做静态检查；完整验证需要把目录放入新月杀 `packages` 后在客户端中开房测试。
